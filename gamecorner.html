Video Game Mobile App

Overview

A 4-in-1 iOS/Android app containing Tic Tac Toe, Mastermind, Video Poker, and a turn-based Battle Simulator, along with respective AI players - all built from scratch.

Link: https://expo.dev/@rlemon17/lemongamecorner

Requires the Expo Development App

Features: React Native, Node.js, Expo, AI Logic, Screen Navigation, Game Development

Home Page

The app's home page presents all 4 available games. The info/confirm box at the bottom corresponds with the currently selected button.

Pressing "Play!" brings the user to that game's settings page, where they can set up the number of human/CPU players, difficulty, and any other options specifically for that game.

React Navigation was used to navigate and render all the different available screens in the app.

Tic Tac Toe

Settings: 1-2 human players, difficulty of AI, whether user is X's or O's

The classic 2-player, three-in-a-row game.

The Easy AI simply chooses a random, open spot. The Hard AI prioritizing winning moves, then stopping any player winning moves.

Players simply tap on the box they wish to play in, and an error message shows if the spot is taken. Scores are kept at the top, and a dialogue box appears each time a round ends, announcing the results.

Mastermind - Hacker

Settings: 1-2 human players, whether user is Codemaker or Hacker, if duplicate colors are allowed

As the Hacker, the user must use the Codemaker's hints to guess the correct, 4-colored code. Pressing on a button on the current active row brings up a small menu to choose that button's color.

Edge cases are accounted for, prompting an error message if a Hacker tries to skip their turn or submit an invalid code (ex. leaving a button blank or having duplicate colors if that setting was turned off)

An AI Hacker starts with a random guess, then uses the Codemaker's hints to make a more educated guess, making sure it doesn't repeat itself or make any decisions that conflict previous Codemaker hints. Its guesses are based on the best hint the Codemaker has given them so far.

Mastermind - Codemaker

As the Codemaker, the user creates the 4 colored code (using a similar interface as the hacker), then rates each of the Hacker's guesses using the smaller interface on the right of each row. Like the Hacker's turns, error checks are put in place if the user tries to input an invalid code.

A Modal pops up upon pressing said interface, asking the user to input the number of dark and light pegs to rate the guess. Error checks are put in place, popping an error if the user tries to submit an incorrect or invalid hint.

The Codemaker may also choose to show the answer, to help determine the hint. The answer is hidden again once the Codemaker completes their turn.

The Codemaker recieves a point for each row in the round, and this is the only time players can receive points.

Once the Hacker discovers the code or 12 rows are played, the winner is declared and the player is prompted to play again (either switching roles or keeping the same roles).

Video Poker

Settings: Dollars per credit (default $0.25), starting money (default $10.00)

Tapping the cards (after the first deal) keeps that card for the second deal, while the rest are shuffled.

The card deck is created and analyzed using a numbered array from 0 to 51. Suits are determined through (n mod 4), and Ranks through (n mod 13), where n is 0 to 51.

The app recognizes all poker hands by calling only 3 functions: checking rank sequences, checking suits, and checking duplicate ranks.

Winning hands cause the upper section to highlight the title, and the lower section to display the earnings.

Error checks prevent any invalid bets based on current credits, and certain touchable interfaces are unusable during some game phases (ex. changing the bet after the first 5 cards are dealt).

Battle Simulator - Character Select

Settings: Whether each of the 4 players are Human or AI, the character of each player

This is a 2v2 turn-based battle game (similar to Pokemon and Paper Mario) revolved around moves that cost mana/energy, and an ultimate move that can only be used below 40% HP. It features character sprites/animations and move animations created by me!

Tapping on a character card displays its sprite, stats (where bar lengths are relative to other character's stats), and moves.

I am still adding new characters to the game, while balancing current characters as I get feedback from testers.

Battle Simulator - Move Selection

Each player chooses their move, then the target(s) for the turn using the bottom interface.

Moves deal damage, heal, apply a status effect (either positive or negative), or a mixture of the three. Moves are programmed individually under one main function and have unique animations.

Impossible choices (ex. not enough mana for a move) and irrational moves (ex. healing an enemy) are greyed out and unselectable.

Each character has it's own unique AI due to their different playstyles and moves. The AIs are programmed to make strategic and informed decisions, rather than just randomly picking an available move and target.

Battle Simulator - Battle Scene

Attacking order is based on character's speed stats. Order is randomized every turn for those with equal speeds.

Dealing or healing damage prompts a number to pop up by the affected character.

Status effects last varying number of turns and include altering stats, damage over time, or being unable to attack. If a status effect exists on a character, they are displayed below their stat bubble.

Defeated characters are no longer animated, greyed out, and untargetble.

The game continues until one side wins!
